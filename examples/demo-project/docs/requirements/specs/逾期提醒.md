# 开发规格:逾期提醒
> spec 版本 v1 · 2026-07-14 · 状态:已确认,可开工

## 功能说明

客户借款单逾期后系统自动发送提醒,替代财务人工翻表核对。
> 证据: docs/requirements/raw/2026-07-10-李姐微信语音转述.md:4 | "现在都靠财务自己翻表,老漏"

## 关键决定(带溯源)

1. **逾期起算**:到期日次日即逾期,无宽限期。【业务确认】(2026-07-14,王芳经李姐电话代确认,回执待王芳补落款,见待办)
   > 证据: docs/requirements/raw/2026-07-14-确认单v1-回执.md:20 | "到期日次日"
2. **部分还款**:只要单据仍有余额,继续提醒。【业务确认】(2026-07-14,同上)
   > 证据: docs/requirements/raw/2026-07-14-确认单v1-回执.md:30 | "只要有余额就继续提醒"
3. **提醒渠道**:站内信。业务未指定渠道,选站内信可绕开短信审批口径,本期不做短信。【开发拟定】(2026-07-14,已列入确认单 v2 送过目)
   > 证据: docs/requirements/raw/2026-07-10-李姐微信语音转述.md:5 | "短信或者站内信都行"
   > 证据: docs/requirements/raw/2026-07-14-确认单v1-回执.md:9 | "营销类短信要审批,业务通知类不用"
4. **提醒频率**:每 3 天一次,最多 3 次,余额清零即停。业务只给方向"别太烦人",细则由开发落定。【开发拟定】(2026-07-14,已列入确认单 v2 送过目)
   > 证据: docs/requirements/raw/2026-07-14-确认单v1-回执.md:26 | "这个你看着办,别太烦人就行"
5. 系统"已有可复用的站内信通道"——【假设】,来自开发者自己的记忆,未查代码核实,开工第一件事验证。

## 输入 / 输出 / 处理逻辑

- 输入:借款单(到期日、余额、客户);每日定时任务扫描。
- 输出:站内信提醒记录(单据号、发送时间、第几次)。
- 逻辑:按决定 1 判定逾期 → 按决定 4 的频率与上限发送 → 按决定 2 的余额条件终止。

## 边界(本期不做)

- 短信渠道(决定 3)。
- 逾期周报表:回执中混入的新需求,已剥离,见 parking.md(不在本 spec 范围)。
  > 证据: docs/requirements/raw/2026-07-14-确认单v1-回执.md:20 | "想要一张逾期情况的周报表"

## 验收标准

- 到期日当天不提醒,次日生成首条提醒(对应决定 1)。
- 部分还款后余额大于零,提醒不中断;余额清零后不再产生提醒(对应决定 2、4)。
- 提醒间隔与次数上限符合决定 4;超上限后停止。
- 全程无短信发出(对应决定 3)。
